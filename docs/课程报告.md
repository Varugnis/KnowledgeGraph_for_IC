## 第一章 问题背景与任务定义

### 1.1 领域选择与背景

本课题选择**半导体与芯片技术**作为知识图谱构建领域。半导体产业是现代信息技术的物质基础，涵盖芯片设计公司、晶圆代工厂、处理器架构、制造工艺、关键材料与器件等大量相互关联的领域知识。近年来，随着大语言模型（LLM）的广泛应用，用户对半导体技术细节的查询需求日益旺盛，而现有大模型在该领域存在明显的事实性错误与知识幻觉问题，如将工艺节点混淆（"4nm左右"）、错误归属代工厂等。

半导体领域适合构建知识图谱的主要理由如下：

1. **实体边界清晰**：公司、芯片、工艺节点、CPU架构、材料、器件、应用场景等类别界定明确；
2. **关系可结构化**：设计关系（designs）、代工关系（fabricates）、工艺采用（uses_process）、继代关系（successor_of）等均可用三元组表示；
3. **推理任务丰富**：多跳路径推理（如"芯片→工艺→材料"）、代际演进链、竞争关系继承等均可在图上实现；
4. **大模型幻觉明显**：工艺型号、代工厂、架构归属等细粒度事实是大模型的高频出错点，适合用图谱进行约束与增强。

### 1.2 任务目标

本次课程作业设定以下三个核心任务：

**任务一（图谱构建）**：构建半导体领域知识图谱，覆盖全球主流芯片厂商、旗舰芯片产品、制造工艺节点、关键材料与应用场景，规模满足100实体、300三元组以上。

**任务二（知识推理）**：实现规则推理与路径推理两种方法，在已有图谱上补全隐含知识（如"台积电代工H100，H100用TSMC 4nm工艺，故台积电掌握4nm工艺"），并支持多跳问题的证据链发现。

**任务三（大模型增强）**：基于GraphRAG方法，将图谱子图检索结果注入大模型Prompt，对比"无图谱直接问答"与"图谱增强问答"在准确性、可追溯性、幻觉抑制、多跳推理四个维度的差异。

### 1.3 大模型在该任务中的不足

当前大语言模型（如 GPT、DeepSeek 等）在半导体知识查询中存在以下典型问题：

- **事实幻觉**：工艺节点描述模糊（"约4nm"）、代工厂归属错误（将苹果芯片的TSMC代工错误描述为Samsung）；
- **缺乏证据**：直接给出结论但无法追溯来源，不适合需要核查的工程场景；
- **多跳推理弱**：面对"A17 Pro和骁龙8 Gen 3哪个工艺更先进"这类需要两步以上推理的问题，容易出现逻辑断层；
- **知识更新滞后**：无法反映最新工艺发布、代工关系变更等动态信息。

知识图谱通过提供结构化、可验证的三元组知识，作为事实约束层与大模型配合，可系统性改善上述问题。

## 第三章 推理方法设计

### 3.1 推理任务说明

本项目实现了**两种推理方法**（作业要求至少一种）：

- **规则推理**：在已有图上通过 Horn 规则前向链式推理，补全隐含的知识三元组；
- **路径推理**：通过双向 BFS 在图上发现实体间的多跳连接路径，用于证据链检索与多跳问答。

辅以**链接预测启发式评分**（共同邻居 + 最短路径 + Jaccard系数），用于实验对比分析。

### 3.2 方法一：规则推理（Horn 规则）

**算法描述**：

规则形式为 `若(A, rel1, B) ∧ (B, rel2, C) 则 (A, new_rel, C)`，置信度为两跳置信度乘以规则衰减系数。共设计7条规则，例如：

| 规则编号 | 前提1 | 前提2 | 推导结论 | 置信度系数 |
|---------|-------|-------|---------|-----------|
| R1 | A designs B | B uses_process C | A company_uses_process C | 0.95 |
| R2 | A fabricates B | B uses_process C | A fab_masters_process C | 0.90 |
| R3 | A successor_of B | B competes_with C | A inherits_competition C | 0.85 |
| R4 | A designs B | B competes_with C | A company_competition_inferred C | 0.80 |
| R5 | A uses_device D | D uses_material M | A process_requires_material M | 0.90 |
| R6 | A used_for X | X requires_device D | A chip_contains_required_device D | 0.75 |
| R7 | A fabricates B | B successor_of C | A fab_advanced_process C | 0.85 |

**输入输出**：
- 输入：NetworkX 有向多重图 G
- 输出：新推导三元组列表 `(主体, 新关系, 客体, 置信度, 描述, 桥接节点)`
- 置信度 ≥ 0.7 的三元组写回图谱（标注 `inferred=True`）

**推理样例**：

规则R1推导示例：
```
已知：(TSMC, fabricates, H100) ∧ (H100, uses_process, TSMC 4nm)
推导：(TSMC, fab_masters_process, TSMC 4nm)  [置信度: 0.90]
含义：TSMC通过代工H100，掌握了TSMC 4nm工艺
```

**实验结果**：

| 规则 | 新增三元组数 | 含义 |
|------|------------|------|
| company_uses_process | 19 | 公司通过旗下芯片间接使用工艺 |
| fab_masters_process | 9 | 代工厂掌握工艺能力 |
| process_requires_material | 28 | 工艺通过器件依赖材料 |
| chip_contains_required_device | 27 | 芯片用途推断所含器件 |
| inherits_competition | 1 | 继代产品继承竞争关系 |
| 共计 | **104** | 置信度≥0.7者写回图谱 |

**错误案例分析**：规则R3（竞争继承）仅推导出1条，因图中同时具备 `successor_of` 和 `competes_with` 两个前提的链条较少，说明图谱三元组覆盖竞争关系的数量尚不足，是后续扩展方向。

### 3.3 方法二：路径推理（双向 BFS）

**算法描述**：

从源实体出发，在有向图上同时遍历正向边（`out_edges`）和反向边（`in_edges`，标注 `inv_` 前缀），BFS 最大跳数 4，最多返回5条路径。

**输入输出**：
- 输入：图 G、起点实体ID、终点实体ID
- 输出：所有满足跳数约束的路径列表，每条路径格式为 `[(节点, 关系, 节点), ...]` 并格式化为可读字符串

**推理样例（多跳路径）**：

问题：AMD与EUV光刻技术有何关联？

```
路径1：[AMD] --designs--> [Ryzen 9 7950X] --uses_process-->
        [TSMC 4nm] --inv_enables_process--> [EUV光刻]
路径2：[AMD] --designs--> [Ryzen 9 7950X] --inv_fabricates-->
        [TSMC] --masters_technology--> [EUV光刻]
```

两条路径分别揭示了 AMD→（工艺依赖）→EUV 和 AMD→（代工厂关系）→EUV 两种推理路线，均有效。

**路径推理准确率评估**：

| 测试用例 | 期望 | 结果 | 正确 |
|---------|------|------|------|
| AMD产品关联EUV技术 | 有路径 | 5条 | ✅ |
| 苹果与台积电的合作关系 | 有路径 | 5条 | ✅ |
| H100到硅材料 | 有路径 | 5条 | ✅ |
| Intel产品关联AI训练 | 有路径 | 5条 | ✅ |
| 麒麟9000S与SMIC工艺 | 有路径 | 5条 | ✅ |
| NVIDIA产品关联EUV | 有路径 | 5条 | ✅ |
| Intel与AMD直接路径 | 无直接路径 | 有（通过竞争关系） | ❌ |

**准确率：85.7%（6/7）**

**错误案例分析**：测试用例7期望两公司之间没有"直接路径"（仅有间接竞争），但 BFS 找到了通过 `competes_with` 的2跳路径，算法判定"有路径"与期望不符。原因在于测试期望设置过于严格——实际上找到竞争路径本身是合理的推理结果，属于评估标准问题而非算法错误。

## 第四章 大模型增强方法

### 4.1 整体框架：GraphRAG

本项目采用 **GraphRAG（Graph Retrieval-Augmented Generation）** 方法，将知识图谱子图作为大模型的外部知识来源，通过 Prompt 注入实现增强问答。与 Text2Cypher 不同，GraphRAG 不需要用户写查询语言，适合自然语言问答场景。

**完整流程如下**：

```
用户自然语言问题
    ↓
① 实体检索（search_entity）
  对问题和关键词做模糊匹配，找出图谱中最相关的Top-5实体
    ↓
② 子图扩展（retrieve_subgraph）
  以匹配实体为种子，BFS扩展2跳，提取局部子图（最多60节点）
    ↓
③ 三元组转文本（subgraph_to_text）
  将子图边转化为自然语言形式：
  "(台积电) --[代工制造]--> (H100)"
    ↓
④ 填入 Prompt 模板（kg_augmented_prompt.txt）
  将三元组文本填入 {kg_context}，用户问题填入 {question}
    ↓
⑤ 调用大模型（DeepSeek API / 模拟模板）
  要求模型标注【KG事实】与【背景知识】并给出置信度
    ↓
⑥ 对比无KG直接问答的回答
```

### 4.2 Prompt 设计

`prompts/kg_augmented_prompt.txt` 中设计了以下约束：

1. **优先使用KG事实**：要求模型用`【KG事实】`标记来自图谱的信息；
2. **标注不确定性**：若图谱无证据，须说明"未找到直接证据"并标注幻觉风险；
3. **结构化输出**：直接答案 → 证据三元组列表 → 置信度评估；
4. **多跳推理**：若问题需多步推理，须显式列出推理链。

### 4.3 幻觉检测

在 `evaluate.py` 中实现了基于图谱的事实核查模块 `simulate_hallucination_check()`：给定一条 LLM 声明，在图谱中查询对应三元组是否存在，输出"事实正确"或"无法验证/可能错误"。

### 4.4 对比实验设计

共设计10个测试问题，覆盖事实查询、一对多关系、比较推理、多跳推理、技术分析等类别，每个问题均进行"无KG"与"有KG"双路问答。

|---------|---------|-----------|---------|

### 5.5 案例分析

**成功案例——多跳比较推理（Q3）**：

- 无KG回答："A17 Pro可能采用3nm工艺，骁龙8 Gen 3也在4nm左右。"（模糊，"可能"）
- KG增强后：检索到 `(A17 Pro, uses_process, TSMC 3nm)` 与 `(骁龙8 Gen 3, uses_process, TSMC 4nm(N4P))`，再通过 `(TSMC 3nm, successor_of, TSMC 4nm)` 推断3nm更先进，给出完整推理链，置信度高。

**成功案例——幻觉检测（案例五）**：

- LLM声明："Apple M3 Ultra采用Samsung 3nm GAA工艺。"
- 图谱查询：不存在该三元组，正确事实为 `(M3 Ultra, uses_process, TSMC 3nm)`。
- 判定：幻觉检出。Apple芯片历来由TSMC代工，该声明属于代工厂归属的典型幻觉。

**失败案例——材料类实体检索（Q10）**：

- 问题："台积电3nm工艺使用了哪些关键材料和技术？"
- 检索结果精确率仅0.20：返回了P001、P002、P003等工艺节点，而非期望的M001（硅）、M007（氧化铪）、T001（EUV）等跨类型实体。
- 原因分析：实体检索基于名称模糊匹配，"台积电"和"3nm"更易命中工艺实体，材料实体距离查询词语义距离较大。

### 5.6 局限性讨论

1. **图谱覆盖不完整**：部分芯片的架构关系、功耗数据等尚未录入，孤立节点5个；
2. **争议信息处理**：麒麟9000S代工厂信息（SMIC vs Samsung）在图谱中存在标注不一致，虽用置信度标注，但仍影响检索精度；
3. **检索跨类型能力弱**：实体检索依赖名称匹配，对材料、技术类实体召回率偏低，需引入语义相似度（如词向量）改善；
4. **规则覆盖有限**：7条规则覆盖的推理模式较少，竞争继承规则（R3）因图谱关系稀疏仅推导出1条；
5. **未实现表示学习推理**：TransE、RotatE等方法可在大规模图上提升链接预测效果，本项目因数据规模和时间限制未实现。
